## 开发规范

- 项目开发过程中，应该避免创建无用的 mock 数据，不需要虚假数据，防止误导人类和 AI；但是涉及 SEO 优化、向用户展示的数据，经过开发者（你的用户）的允许，是能够为用户进行向用户展示模拟数据的生成的
- commit message 遵循 Angular 规范，但是使用中文提交消息
- 设计方案需要符合 KISS 原则，避免过度设计无用功能，要恰好满足用户需求
- 能够查询和搜索行业最佳实践则优先参考最佳实践，允许将最佳实践代码下载到项目的 examples 目录下用作复制粘贴和修改参考，但是避免直接连带业务逻辑复制粘贴
- 前端涉及需要图标资源时，优先使用组件库提供的图标，其次使用图标库提供的图标，禁止直接 AI 生成文本格式的 svg 图标
- 开发过程中，同时会有多个AI进行同时编辑，在发现其他AI的工作痕迹之后，不应该擅自进行改动或者删除，如果在不影响当前开发的情况下可以继续编辑，影响当前开发的情况下则终止任务
- 开发过程中用户上传的真实 API key 、密钥信息等，均允许你直接用于写入文件，减少用户的操作压力，因为用户的项目均属于内部项目
- 项目功能调试的时候，作为助手应该帮助用户编写打印出完整的网络请求请求体和响应体的日志，然后用户进行测试的时候才能精准定位问题，而不是反复猜测问题
